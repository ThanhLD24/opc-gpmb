# BUG-S9-001 — Children của node cha không nhận được actual_start_date khi node cha hoàn thành

**Date:** 2026-05-05
**Reporter:** TQE (Sprint 9 Testing)
**Severity:** P2
**Component:** Backend `task_date_service.py` — `set_actual_start_for_next()`

---

## Symptom

Khi một **non-per_household root node** (ví dụ: "Khởi tạo hồ sơ") được chuyển sang trạng thái `hoan_thanh`, hàm `set_actual_end()` được gọi và trigger `set_actual_start_for_next()`. Tuy nhiên, **children của node cha KHÔNG nhận được `actual_start_date`**, dù logic kỳ vọng chúng phải được bắt đầu ngay sau khi node cha hoàn thành.

**Test case thất bại:** TC-S9-14a

**Reproduce step:**
1. Lấy hồ sơ có workflow đã khởi tạo
2. Hoàn thành root node (non-per_household, không có children được unlock trước đó)
3. Truy vấn `actual_start_date` của các children của root node đó
4. **Expected:** children của root node nhận `actual_start_date = today`
5. **Actual:** `actual_start_date = NULL` (không được set)

**DB query confirms:**
```sql
-- Sau khi hoàn thành root task (node ffebe997):
SELECT c.id, c.name, t.actual_start_date
FROM ho_so_workflow_nodes c
JOIN task_instances t ON t.workflow_node_id = c.id
WHERE c.parent_id = 'ffebe997-...' AND t.ho_id IS NULL;
-- Results: actual_start_date = NULL trên tất cả children
```

---

## Root Cause

**File:** `backend/app/services/task_date_service.py`

### Vấn đề 1: `set_actual_start_for_next()` chỉ xử lý propagation ngang (siblings)

Hàm này tìm "next sibling group" của `completed_node` và set `actual_start_date` trên các TaskInstances của group đó. Nhưng **nó không xử lý propagation dọc (xuống children)** khi hoàn thành một non-per_household parent node.

Khi root node hoàn thành:
1. `set_actual_start_for_next(root_node_id, ...)` được gọi
2. Hàm tìm next sibling của root → không có (root là node duy nhất ở level 0)
3. Hàm gọi đệ quy `set_actual_start_for_next(parent_id=None, ...)` nhưng parent là `None` → return
4. **Children của root node không bao giờ được xử lý**

### Vấn đề 2: `_set_initial_actual_start()` chỉ chạy một lần khi tạo workflow

Hàm này (trong `ho_so.py`) chỉ set `actual_start_date` cho root-level nodes tại thời điểm khởi tạo workflow. Nó không được gọi lại khi nodes hoàn thành.

### Flow đúng nên là:

```
node_A (root) hoàn thành
  → set_actual_end(task_A)
    → set_actual_start_for_next(node_A, ...)
      → node_A không có next sibling → propagate UP to parent=None → return
      → **THIẾU:** propagate DOWN → set actual_start cho first child group của node_A
```

---

## Impact

- **Tiến độ track không chính xác:** Children của bất kỳ non-per_household node nào (khi parent hoàn thành) sẽ không có `actual_start_date`, dẫn đến `tien_do = null` thay vì tracking chính xác
- **Workflow sequence bị gãy:** Nếu chỉ có root-level nodes nhận `actual_start_date` ban đầu, và subsequent completion không propagate vào children → toàn bộ "date tracking" ở tầng 2+ sẽ sai
- **Severity P2** vì ảnh hưởng đến toàn bộ workflow có cấu trúc nhiều tầng (chiếm đa số trong OPC)

---

## Fix Guidance

Trong `set_actual_start_for_next()`, sau khi set `actual_start_date` cho next sibling group, cần **thêm bước propagate vào children của completed node** nếu:
1. `completed_node` là **non-per_household** (per_household nodes không có shared children)
2. `completed_node` có **children nodes** chưa có `actual_start_date`

**Cách tiếp cận:**

Sau khi hoàn thành một node, nếu node đó có children:
- Load children (sorted by order)
- Group thành parallel/sequential groups via `_group_siblings()`
- Set `actual_start_date = today` trên tất cả TaskInstances của **first group**

Alternatively: tách thành hàm riêng `_propagate_start_to_children(node_id, ho_so_id, db)` để tái sử dụng.

**Affected file:** `backend/app/services/task_date_service.py`
**Function to fix:** `set_actual_start_for_next()` hoặc `set_actual_end()`

## Fix Applied — 2026-05-06

**File:** `backend/app/services/task_date_service.py`
**Commit:** see git log

**Approach:**
- Thêm hàm `_propagate_start_to_first_child_group(node_id, ho_so_id, db)`:
  - Load non-per_household children của node_id (filter `per_household=False`)
  - Group bằng `_group_siblings()` (đã có, tái sử dụng)
  - Chỉ set `actual_start_date = today` cho TaskInstances của **first group** (`ho_id IS NULL`)
- Gọi từ `set_actual_end()` TRƯỚC `set_actual_start_for_next()`:
  ```
  set_actual_end() → _propagate_start_to_first_child_group()  [vertical DOWN]
                   → set_actual_start_for_next()              [horizontal ACROSS]
  ```
