# BUG-S6-001 — Filter dropdowns empty on /ho-dan và /cong-viec

**Severity:** P2 — Major (feature broken but workaround exists: hồ sơ detail still accessible via /ho-so-gpmb)  
**Sprint:** 6  
**Reporter:** Tester QE  
**Date:** 2026-04-30  
**Status:** 🔴 OPEN

---

## Summary

Tất cả filter dropdown (Hồ sơ, Trạng thái, CBCQ) trên trang `/ho-dan` và filter Hồ sơ trên `/cong-viec` Tab 1 đều **không load được dữ liệu** — dropdown rỗng sau khi trang load xong.

---

## Reproduction Steps

1. Đăng nhập với bất kỳ role nào
2. Vào `/ho-dan` (Quản lý Hộ dân)
3. Click vào dropdown "Tất cả hồ sơ"
4. **Kết quả:** Dropdown rỗng, không có option nào
5. Tương tự: vào `/cong-viec` → Tab 1 → click dropdown "Chọn hồ sơ GPMB" → rỗng

---

| | Expected | Actual |
|--|----------|--------|
| Dropdown "Tất cả hồ sơ" | Hiển thị 4 hồ sơ (HS-202504-001..004) | Rỗng |
| Dropdown "Tất cả CBCQ" | Hiển thị Nguyễn Văn Khánh | Rỗng |

## Root Cause

**Frontend** gọi `/ho-so?page_size=200` để load danh sách hồ sơ cho filter.

**Backend** endpoint `GET /ho-so` có constraint: `page_size: int = Query(20, ge=1, le=100)` — max page_size là **100**.

Kết quả: request với `page_size=200` trả về **422 Unprocessable Entity** → React Query nhận lỗi → `hoSoList` undefined → `hoSoOptions = []`, `cbcqOptions = []`.

```
[ERROR] Failed to load resource: 422 (Unprocessable Entity)
@ http://localhost:8000/api/v1/ho-so?page_size=200
```

**Affected file:** `backend/app/api/v1/ho_so.py`, line 148

```python
page_size: int = Query(20, ge=1, le=100)  # ← le=100 là nguyên nhân
```

**Affected FE calls:**
- `frontend/src/app/(dashboard)/ho-dan/page.tsx` — `api.get('/ho-so?page_size=200')`  
- `frontend/src/app/(dashboard)/cong-viec/page.tsx` — `api.get('/ho-so?page_size=200')`

---

## Impact

- `/ho-dan` filter bar hoàn toàn vô dụng (3 filters đều rỗng)
- `/cong-viec` Tab 1 không thể chọn hồ sơ → không xem được công việc
- Regression trên các màn khác nếu có dùng cùng pattern (cần confirm)

---

## Recommended Fix

**Option A — Fix BE (preferred):** Tăng giới hạn `le=100` lên `le=500` trong `ho_so.py` line 148. FE giữ `page_size=200`.

```python
# Before:
page_size: int = Query(20, ge=1, le=100)
# After:
page_size: int = Query(20, ge=1, le=500)
```

**Option B — Fix FE only:** Đổi `page_size=200` → `page_size=100` trong cả 2 file FE. Nhược điểm: nếu số hồ sơ > 100 trong tương lai thì filter vẫn thiếu.

**Recommendation:** Option A — vì filter cần load ALL hồ sơ để đủ option, BE limit nên cao hơn FE request.

---

## Fix Effort

Rất nhỏ — 1 dòng thay đổi trong BE. Không cần migration, không impact API contract.
